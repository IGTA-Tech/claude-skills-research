# 贡献指南 / Contributing Guide

感谢你对 Claude Skills Guide 项目的关注！我们欢迎所有形式的贡献。

Thank you for your interest in contributing to Claude Skills Guide! We welcome all forms of contributions.

---

## 🌟 如何贡献 / How to Contribute

### 1. 报告问题 / Report Issues

如果你发现了文档错误、不清楚的说明或有改进建议：

If you find documentation errors, unclear instructions, or have suggestions:

1. 在 [Issues](https://github.com/sanshao85/claude-skills-guide/issues) 页面创建新 issue
2. 清楚地描述问题或建议
3. 如果可能，提供具体的改进方案

### 2. 改进文档 / Improve Documentation

文档改进是最受欢迎的贡献！

Documentation improvements are highly appreciated!

**可以改进的内容 / What you can improve:**
- 修正错别字和语法错误 / Fix typos and grammar
- 改进示例代码 / Improve code examples
- 添加更清晰的说明 / Add clearer explanations
- 补充缺失的内容 / Add missing content
- 改进翻译 / Improve translations

**步骤 / Steps:**
1. Fork 本仓库
2. 创建新分支：`git checkout -b improve-docs`
3. 进行修改
4. 提交更改：`git commit -m "docs: improve XXX section"`
5. 推送分支：`git push origin improve-docs`
6. 创建 Pull Request

### 3. 分享你的 Skill / Share Your Skills

如果你创建了优秀的 Skill 示例，欢迎分享！

If you've created excellent Skill examples, please share!

**要求 / Requirements:**
- 遵循最佳实践 / Follow best practices
- 包含清晰的文档 / Include clear documentation
- 提供使用示例 / Provide usage examples
- 代码质量良好 / Good code quality

**步骤 / Steps:**
1. 在 `community-skills/` 目录下创建你的 Skill
2. 添加 README 说明
3. 提交 Pull Request

### 4. 翻译 / Translations

我们欢迎将文档翻译成其他语言！

We welcome translations to other languages!

**当前语言 / Current languages:**
- 🇨🇳 中文（主要）/ Chinese (Primary)
- 🇺🇸 英文（部分）/ English (Partial)

**需要翻译 / Needed:**
- 🇯🇵 日文 / Japanese
- 🇰🇷 韩文 / Korean
- 🇪🇸 西班牙文 / Spanish
- 🇫🇷 法文 / French

## 📝 提交规范 / Commit Convention

请使用语义化的提交信息：

Please use semantic commit messages:

- `docs:` - 文档更新 / Documentation updates
- `feat:` - 新功能 / New features
- `fix:` - 修复错误 / Bug fixes
- `style:` - 格式调整 / Style changes
- `refactor:` - 重构 / Refactoring
- `test:` - 测试相关 / Testing
- `chore:` - 其他杂项 / Other chores

**示例 / Examples:**
```bash
docs: 修正快速开始章节的错别字
docs: fix typos in quick start section

feat: 添加新的 Skill 示例
feat: add new skill example

fix: 修复代码示例中的错误
fix: correct error in code example
```

## ✅ Pull Request 检查清单 / PR Checklist

提交 PR 前，请确保：

Before submitting a PR, please ensure:

- [ ] 代码/文档遵循项目风格 / Code/docs follow project style
- [ ] 所有链接都有效 / All links are valid
- [ ] 示例代码可以运行 / Example code works
- [ ] 提交信息清晰明确 / Commit messages are clear
- [ ] 已阅读贡献指南 / Have read contributing guide

## 🤝 行为准则 / Code of Conduct

### 我们的承诺 / Our Pledge

为了营造开放和友好的环境，我们承诺：

To foster an open and welcoming environment, we pledge to:

- 尊重不同的观点和经验 / Respect differing viewpoints and experiences
- 接受建设性的批评 / Accept constructive criticism gracefully
- 关注对社区最有利的事情 / Focus on what is best for the community
- 对其他社区成员表示同理心 / Show empathy towards other community members

### 不可接受的行为 / Unacceptable Behavior

- 使用性化的语言或图像 / Use of sexualized language or imagery
- 人身攻击或侮辱性评论 / Personal attacks or insulting comments
- 骚扰行为 / Harassment of any kind
- 发布他人的私人信息 / Publishing others' private information
- 其他不道德或不专业的行为 / Other unethical or unprofessional conduct
