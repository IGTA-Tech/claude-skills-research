## 🎯 核心概念

### Skills 是什么？
标准化的能力包，将专家知识封装成可复用模块，让 Claude 成为领域专家。

### 三层结构
```
skill-name/
├── SKILL.md          # SOP - 标准作业程序
├── scripts/          # Tools - 可执行脚本
└── reference/        # Resources - 参考资料
```

### 渐进式披露
- **Level 1**: name + description (~100 tokens) - 始终加载
- **Level 2**: SKILL.md 主体 (~5k tokens) - 触发时加载
- **Level 3**: scripts + reference - 按需加载

## 📝 SKILL.md 模板

```yaml
---
name: skill-name                    # 小写+连字符，最大64字符
description: 做什么 + 何时用 + 触发词  # 最大1024字符
---

# Skill Title

简短概述（1-2句话）

## Capabilities
- 能力1
- 能力2

## How to Use
1. 步骤1
2. 步骤2

## Example Usage
"示例请求1"
"示例请求2"

## Scripts
- `script.py`: 描述

## Limitations
- 限制1
```

## 🐍 Python 脚本模板

```python
#!/usr/bin/env python3
import sys
import json

def main():
    try:
        # 读取输入
        input_data = json.loads(sys.stdin.read())
        
        # 处理数据
        result = {"status": "success", "data": input_data}
        
        # 输出结果
        print(json.dumps(result, indent=2))
        
    except Exception as e:
        error = {
            "status": "error",
            "message": str(e)
        }
        print(json.dumps(error), file=sys.stderr)
        sys.exit(1)

if __name__ == "__main__":
    main()
```

## ✅ 最佳实践清单

### Description 编写
- [ ] 包含"做什么"和"何时用"
- [ ] 包含关键触发词
- [ ] 精准，避免过度触发
- [ ] 最大 1024 字符

### SKILL.md 编写
- [ ] 保持简洁（< 500 行）
- [ ] 只教 Claude 不知道的
- [ ] 提供具体示例
- [ ] 使用清晰的步骤
- [ ] 详细文档放 reference/

### 脚本开发
- [ ] 完善的错误处理
- [ ] JSON 输入/输出
- [ ] 清晰的文档字符串
- [ ] 输入验证
- [ ] 调试信息输出到 stderr

### 测试
- [ ] 在 Haiku、Sonnet、Opus 上测试

- [ ] 测量 token 使用

- [ ] 使用正斜杠 `/` 路径

## 🚫 常见错误

### ❌ 不要做
```yaml
# 太简短
description: Financial tool

# 太泛化
description: Helps with data

# 第一人称
description: I can analyze data

# Windows 路径
reference\guide.md

# 过度解释
Python is a programming language...
```

### ✅ 应该做
```yaml
# 精准描述
description: Calculate financial ratios (ROE, ROA, P/E) from statements. Use when analyzing company performance.

# Unix 路径
reference/guide.md

# 简洁直接
Use this formula: ROE = Net Income / Equity
```

## 📊 设计模式速查

### 工作流模式
```markdown

- [ ] 验证输入

### Phase 2: Processing
- [ ] 处理数据

### Phase 3: Validation
- [ ] 运行验证脚本
```


## 🐛 故障排除

### Skill 未触发
- 检查 description 是否包含触发词
- 验证 YAML frontmatter 格式
- 测试用户请求是否匹配

### 脚本执行失败
```bash
# 检查权限
chmod +x scripts/script.py

# 独立测试
echo '{}' | python scripts/script.py

# 检查依赖
pip list | grep package_name
```

### 输出格式错误
```python
# 正确：只输出 JSON 到 stdout
print(json.dumps(result))

# 正确：调试信息到 stderr
print("[DEBUG]", file=sys.stderr)

# 错误：混合输出
print("Debug info")  # 会破坏 JSON
print(json.dumps(result))
```

## 📦 文件下载

```python
# 提取 file_id
import re
file_ids = re.findall(r'file_[a-zA-Z0-9]+', str(response))

# 下载文件
for file_id in file_ids:
    metadata = client.beta.files.retrieve_metadata(file_id=file_id)
    content = client.beta.files.download(file_id=file_id)
    
    with open(f"outputs/{metadata.filename}", "wb") as f:
        f.write(content.read())
```
