# 🌍 国际化 (Internationalization)

## 语言加载流程

**每个命令执行前必须先加载语言配置：**

### 1. 读取配置文件

```
使用 Read 工具读取 `.claude/locale-config.json`
```

- 成功：继续步骤 2
- 失败（文件不存在）：使用默认配置
  ```json
  {
    "language": "zh-CN",
    "fallback": "zh-CN"
  }
  ```

### 2. 解析语言设置

从配置文件提取：
- `language`: 当前使用的语言代码（如 `"en-US"`, `"zh-CN"`）
- `fallback`: 备用语言代码（翻译缺失时使用）

### 3. 加载语言文件

使用 Read 工具并行读取以下文件：

```
locales/{language}/messages.json
locales/{language}/skill.json
locales/{language}/templates.json
locales/{language}/hooks.json
```

将 JSON 内容解析并存储为 `LANG` 对象，结构如下：

```javascript
LANG = {
  messages: { /* messages.json 内容 */ },
  skill: { /* skill.json 内容 */ },
  templates: { /* templates.json 内容 */ },
  hooks: { /* hooks.json 内容 */ }
}
```

### 4. 翻译文本引用

使用路径引用模式访问翻译文本：

| 引用路径 | 说明 | 示例值（zh-CN） | 示例值（en-US） |
|---------|------|----------------|----------------|
| `LANG.messages.commands.initIndex.confirmDirectory` | 确认目录提示 | "当前目录是 {directory}, 确认这是项目根目录吗？" | "Current directory is {directory}, confirm this is the project root?" |
| `LANG.messages.commands.initIndex.scanning` | 扫描提示 | "正在扫描项目..." | "Scanning project..." |
| `LANG.messages.commands.initIndex.complete` | 完成消息 | "✅ 完成！" | "✅ Complete!" |
| `LANG.templates.common.autoUpdateNotice` | 自动更新声明 | "本注释在文件修改时自动更新..." | "This comment is automatically updated..." |

### 5. 参数替换

翻译文本中的占位符需要替换为实际值：

**占位符格式**: `{变量名}`

**常用占位符**:
- `{directory}` - 目录路径
- `{count}` - 数量
- `{file}` - 文件名
- `{language}` - 语言名称
- `{folders}` - 文件夹数量
- `{nodes}` - 节点数量
- `{time}` - 时间

**替换示例**:
```
模板: "当前目录是 {directory}, 确认这是项目根目录吗？"
变量: directory = "/home/user/my-project"
结果: "当前目录是 /home/user/my-project, 确认这是项目根目录吗？"
```

### 6. 错误处理

**语言文件缺失**:
1. 尝试加载主语言 (`language`)
2. 失败 → 尝试加载备用语言 (`fallback`)
3. 再失败 → 使用硬编码的中文文本（向后兼容）

**JSON 解析错误**:
- 记录错误但继续执行
- 使用 fallback 语言或默认文本

### 7. 使用示例

#### 示例 1: 询问用户确认目录

```markdown
**引用**: LANG.messages.commands.initIndex.confirmDirectory

**实现**:
```
询问用户: 使用 LANG.messages.commands.initIndex.confirmDirectory,
         将 {directory} 替换为当前工作目录路径
```

**输出（zh-CN）**: "当前目录是 /home/user/project, 确认这是项目根目录吗？"
**输出（en-US）**: "Current directory is /home/user/project, confirm this is the project root?"
```

#### 示例 2: 显示扫描结果

```markdown
**引用**: LANG.messages.commands.initIndex.found

**实现**:
```
输出: LANG.messages.commands.initIndex.found
输出: "- " + LANG.messages.commands.initIndex.fileCount (替换 {language} 和 {count})
```

**输出（zh-CN）**:
```
发现:
- JavaScript/TypeScript: 45 文件
- Python: 12 文件
```

**输出（en-US）**:
```
Found:
- JavaScript/TypeScript: 45 files
- Python: 12 files
```
```

#### 示例 3: 生成文件头注释

```markdown
**引用**: LANG.templates.common.autoUpdateNotice

**实现**:
```javascript
/**
 * Input: {依赖列表}
 * Output: {导出内容}
 * Pos: {定位}
 *
 * ${LANG.templates.common.autoUpdateNotice}
 */
```

**输出（zh-CN）**:
```javascript
/**
 * Input: lodash, ./utils
 * Output: createUser(), User
 * Pos: 业务层-用户服务
 *
 * 本注释在文件修改时自动更新，同时触发 FOLDER_INDEX 和 PROJECT_INDEX 更新
 */
```

**输出（en-US）**:
```javascript
/**
 * Input: lodash, ./utils
 * Output: createUser(), User
 * Pos: Business Layer-User Service
 *
 * This comment is automatically updated when the file is modified, triggering FOLDER_INDEX and PROJECT_INDEX updates
 */
```
```

## 快速参考

### 常用翻译路径

| 场景 | 引用路径 |
|------|---------|
| 确认目录 | `LANG.messages.commands.initIndex.confirmDirectory` |
| 扫描项目 | `LANG.messages.commands.initIndex.scanning` |
| 完成提示 | `LANG.messages.commands.initIndex.complete` |
| 更新通知 | `LANG.messages.notifications.indexUpdated` |
| 文件头自动更新 | `LANG.templates.common.autoUpdateNotice` |
| 文件夹自指声明 | `LANG.messages.folderIndex.selfReference` |
| 项目自指声明 | `LANG.messages.projectIndex.selfReference` |

### 配置文件示例

**切换到英文**:
```json
{
  "language": "en-US",
  "fallback": "zh-CN"
}
```

**保持中文（默认）**:
```json
{
  "language": "zh-CN",
  "fallback": "zh-CN"
}
```
